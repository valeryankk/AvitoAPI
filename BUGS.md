## BUG-01
 — Сервер принимает объявление с нулевой ценой

**Краткое описание:** POST /api/1/item принимает `price: 0` и возвращает 200, хотя нулевая цена не является валидной.

**Шаги воспроизведения:**
1. Отправить POST запрос на `/api/1/item`
2. В теле передать `"price": 0`, остальные поля валидные

**Фактический результат:** `HTTP 400 Bad Request`  
**Ожидаемый результат:** `HTTP 200 OK` — нулевая цена валидна (бесплатное объявление)  
**Серьёзность:** Normal

## BUG-08 — Сервер отклоняет объявление с нулевой статистикой

**Краткое описание:** POST /api/1/item возвращает 400 если все поля статистики равны нулю (`likes: 0, viewCount: 0, contacts: 0`), хотя нулевые значения валидны для нового объявления.

**Шаги воспроизведения:**
1. Отправить POST запрос на `/api/1/item`
2. В теле передать `"statistics": {"likes": 0, "viewCount": 0, "contacts": 0}`, остальные поля валидные

**Фактический результат:** `HTTP 400 Bad Request`, `"message": "поле likes обязательно"`  
**Ожидаемый результат:** `HTTP 200 OK` — нулевые значения валидны для нового объявления  
**Серьёзность:** Critical
