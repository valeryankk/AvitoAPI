## TC-01 — POST /api/1/item — Создать объявление
 
### Позитивные сценарии
 
| ID | Название | Входные данные | Ожидаемый результат |
|----|----------|---------------|-------------------|
| TC-01-01 | Создать объявление со всеми полями | Валидные `sellerID`, `name`, `price`, `statistics` | 200, в ответе уникальный `id`, все поля совпадают с отправленными |
 
### Негативные сценарии
 
| ID | Название | Входные данные | Ожидаемый результат |
|----|----------|---------------|-------------------|
| TC-01-03 | Создать объявление без поля `name` | Все поля кроме `name` | 400 Bad Request |
| TC-01-04 | Создать объявление без поля `price` | Все поля кроме `price` | 400 Bad Request |
| TC-01-05 | Создать объявление без поля `sellerID` | Все поля кроме `sellerID` | 400 Bad Request |
| TC-01-06 | Отправить пустое тело запроса | `{}` | 400 Bad Request |
| TC-01-07 | Передать `price` как строку | `"price": "тысяча"` | 400 Bad Request |
| TC-01-08 | Передать `sellerID` как строку | `"sellerID": "abc"` | 400 Bad Request |
 
### Корнер-кейсы
 
| ID | Название | Входные данные | Ожидаемый результат |
|----|----------|---------------|-------------------|
| TC-01-09 | Цена равна нулю | `"price": 0` | 200 |
| TC-01-10 | Отрицательная цена | `"price": -100` | 400 Bad Request *¹* |
| TC-01-11 | Очень длинное `name` (1000 символов) | `"name": "a" * 1000` | 400 Bad Request *¹* |
| TC-01-12 | `contacts` больше `viewCount` | `contacts: 100, viewCount: 1` | 400 Bad Request (бизнес-логика) *¹* |
| TC-01-13 | Отрицательный `sellerID` | `"sellerID": -1` | 400 Bad Request *¹* |
| TC-01-14 | Отрицательный `viewCount` | `"viewCount": -1` | 400 Bad Request *¹* |
| TC-01-15 | Отрицательный `contacts` | `"contacts": -1` | 400 Bad Request *¹* |
| TC-01-16| Все показатели статистики равны нулю | `likes: 0, viewCount: 0, contacts: 0` | 200 (баг — сервер возвращает 400) |

 
### Нефункциональные
 
| ID | Название | Ожидаемый результат |
|----|----------|-------------------|
| TC-01-17 | Время ответа | Ответ приходит менее чем за 2000 мс |
| TC-01-18 | Content-Type ответа | Заголовок `Content-Type: application/json` |
